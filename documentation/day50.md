# Machine Test AI Design – Zecpath

## Objective
To evaluate candidate real-world technical performance through practical tasks.

## Key Features
* **Real code execution**: Submits candidate logic blocks to execution runner checkers.
* **Multi-metric scoring**: Rates correctness, runtime efficiency, formatting checks, and attempt numbers.
* **Time-based evaluation**: Incentivizes fast problem solving by reducing weights on long elapsed times.
* **Practical skill assessment**: Elevates screening from conceptual theory to real coding capability.

## Advantages
* **Real skill validation**: Highlights candidates who can write compiling code under time limits.
* **Objective scoring**: Test suite execution returns consistent, measurable results.
* **Scalable evaluation**: Runs sandboxed verification rules automatically.

## Limitations
* Does not review deep code patterns (e.g. tracking specific algorithm structures).
* Limited optimization diagnostics beyond simple execution runtimes.

## Future Improvements
* Integrated AI code reviewer giving architectural design feedback.
* Plagiarism checks compared against web repositories.
* Real-time coding session replays.
